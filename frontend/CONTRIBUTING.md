# フロントエンド CONTRIBUTING ガイド

このドキュメントでは、フロントエンド開発における CSS/UI の方針をまとめています。

---

## CSS・UI 方針

### 1. Tailwind CSS を基本とする

- React プロジェクトでは Tailwind CSS をメインで使用します。
- ユーティリティクラスを使って、シンプルなスタイル付けを行います。
- Tailwind の知識が十分でない場合でも、既存コードを参考にスタイルを追加してください。

### 2. 複雑なコンポーネント

- 複雑な UI（ナビゲーションバー、モーダルなど）は UI ライブラリを使用して構築しても構いません。
- 当面は [react-bootstrap](https://react-bootstrap.github.io/) を使用します。
- shadcn-ui は現状では使用しません。

### 3. カスタマイズ方針

- なるべく抽象化されているコンポーネントや規約に従ったスタイルを使用してください。
- 過度なカスタマイズは避け、コードの可読性・保守性を優先します。

### 4. デザインの考え方

- デザインの美しさよりも、ユーザーに分かりやすく、使いやすいことを優先します。
- Tailwind を基本にして、必要に応じて react-bootstrap のスタイルを併用します。

---

この方針に従うことで、フロントエンドコードの一貫性を保ち、将来的なメンテナンスを容易にします。

## コード・設計方針

### 1. コンポーネント構造

- フロントエンドは **feature-based ディレクトリ構成** を推奨します。
- UI コンポーネントは **小さく、単一の責務を持つこと** を意識してください。
- 複雑なコンポーネントは **子コンポーネントに分割して再利用可能にする**。

### 2. 型・Props の扱い

- TypeScript を使用し、Props や state には明示的な型を付与してください。
- `any` はなるべく避け、どうしても必要な場合はコメントで理由を明記します。

### 3. Context とフックの使用方針

- 状態管理には必要に応じて Context を使用します。
- Provider には state と操作関数（例：`login`, `logout`）をまとめて渡すことが一般的です。
- Context は **必要最小限の情報に絞り、肥大化させない** ようにしてください。
- カスタムフックを作成して、Context の値や操作関数を簡単に取得できるようにしてください。

### 4. import の統一

- プロジェクト内では **基本的に絶対インポート** を使用します。
- ただし、`index.ts` 内の export は同階層のファイルをまとめる用途なので、**相対インポートで統一して問題ありません**。
- 例：
  ```ts
  // context/index.ts
  export { AuthProvider } from "./AuthProvider";
  export { AuthContext } from "./AuthContext";
  export { useAuthContext } from "./useAuthContext";
  ```
